## Purpose

This file translates the Elasticsearch field names into human-readable English and explains what each field means. It is intended as a source file for another agent/skill that needs to understand Danish Motor Register data.

## Source and confidence notes

The field descriptions below combine:

1. Public Motorstyrelsen context: Motorregistret contains information about vehicles registered in Denmark and exposes categories such as make/model/variant, vehicle category/use, condition, engine, body, equipment, weight, fuel consumption, inspection, insurance, and permissions/exemptions.
2. Public Danmarks Statistik DMR-derived variable descriptions for many core variables such as passenger count, maximum power, total weight, fuel type, registration number, registration status, body type, emissions, first registration date, and more.
3. Inferred translations from the Danish field names for fields that are not directly documented in the public sources used here.

Confidence values:
- `official/inferred`: the field or close DMR variable is described in public DMR-derived documentation, with path-specific interpretation inferred from this Elasticsearch mapping.
- `inferred`: translated from Danish naming conventions and domain context; verify with sample values before using for compliance-critical reporting.
- `system`: Elasticsearch/system metadata, not a vehicle business field.

## Agent rules for this dataset

1. Prefer exact keyword fields for strings: use `field.keyword` for equality filters, `STATS ... BY`, terms aggregations, and sorting.
2. Use numeric/date/boolean fields directly.
3. Do not aggregate object container fields directly.
4. Treat fields ending in `Nummer`, `TypeNummer`, or `Ident` as categorical identifiers/codes unless the description says otherwise.
5. Weight fields are normally kg; power is normally kW; displacement is normally ccm/cm³; odometer is km; CO2 is often g/km. Verify with sample values before formal reporting.
6. Use derived fields when available for common dashboarding:
   - `_drivkraft_primaer` = primary fuel/propulsion
   - `_maale_norm_primaer` = primary measurement norm
   - `_km_per_liter_primaer` = primary km/l
   - `_co2_g_per_km_primaer` = primary CO2 g/km
   - `_foerste_registrering_aar` = first registration year
7. Avoid exposing raw `RegistreringNummerNummer` and `StelNummer` in public/shared dashboards.
8. For car-only dashboards, a common base filter is:
   ```esql
   FROM dmr-raw-000002
   | WHERE KoeretoejRegistreringStatus.keyword == "Registreret"
     AND KoeretoejArtNavn.keyword == "Personbil"
   ```
